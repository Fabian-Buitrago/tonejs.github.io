Tone

Tone is the base class of all other classes. It provides a lot of methods and functionality to all classes that extend it.

new Tone ([inputs][, outputs])
inputs = 1 {number}

the number of input nodes

outputs = 1 {number}

the number of output nodes

.set (params[, value][, rampTime])
params {Object|string}
value {number}
rampTime {Time}
returns {Tone}

this

Set the parameters at once. Either pass in an object mapping parameters to values, or to set a single parameter, by passing in a string and value. The last argument is an optional ramp time which will ramp any signal values to their destination value over the duration of the rampTime.

code
//set values using an object
filter.set({
	"frequency" : 300,
	"type" : highpass
});
code
filter.set("type", "highpass");
code
//ramp to the value 220 over 3 seconds. 
oscillator.set({
	"frequency" : 220
}, 3);
.get (params)
params {Array|string|undefined}
the parameters to get, otherwise will return all available.
returns {Object}

Get the object's attributes. Given no arguments get will return all available object properties and their corresponding values. Pass in a single attribute to retrieve or an array of attributes. The attribute strings can also include a "." to access deeper properties.

code
osc.get();
//returns {"type" : "sine", "frequency" : 440, ...etc}
code
osc.get("type");
//returns { "type" : "sine"}
code
//use dot notation to access deep properties
synth.get(["envelope.attack", "envelope.release"]);
//returns {"envelope" : {"attack" : 0.2, "release" : 0.4}}

.defaultArg (given, fallback)
given {*}
fallback {*}
returns {*}

If the given parameter is undefined, use the fallback. If both given and fallback are object literals, it will return a deep copy which includes all of the parameters from both objects. If a parameter is undefined in given, it will return the fallback property.

WARNING: if object is self referential, it will go into an an infinite recursive loop.

.optionsObject (values, keys[, defaults])
values {Array}

defaults {Object}
optional defaults to mixin to the returned options object
returns {Object}

the options object with the names mapped to the arguments

returns the args as an options object with given arguments mapped to the names provided.

if the args given is an array containing only one object, it is assumed that that's already the options object and will just return it.

.setContext (ctx)
ctx {AudioContext}
The new audio context to set

Tone automatically creates a context on init, but if you are working with other libraries which also create an AudioContext, it can be useful to set your own. If you are going to set your own context, be sure to do it at the start of your code, before creating any objects.
